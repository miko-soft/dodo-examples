Controller Lifecycle Hooks

Every time a new route is opened the controller is changed and new controller lifecycle starts.
This example shows in which order will be executed controller constructor and the lifecycle hooks. Lifecycle hooks are controller methods: __loader, __init, __rend, __postrend and __destroy
Open the browser console to see the constructor and hook arguments.
  1. constructor(app) - Will be executed when browser page is reloaded.
  2. __loader(trx) - Load views via dd-inc and dd-view, load css, js, define title, keywords, description, etc.
  3. __init(trx) - Good place to initialise controller variables and fetch the API data via this.$httpClient.
  4. __rend(trx) - This method shouldn't be used unless you want to improve the app speed.
  5. __postrend(trx) - Do some things after the HTML is erndered and DOM creation is finished.
  6. __destroy(trx) - Activated when a new route is opened and old controller died.
app is the App class instance
trx is the Router transitional variable
