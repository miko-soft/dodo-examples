Controller Lifecycle Hooks

Every time a new route is opened, the current controller is replaced and a new controller lifecycle begins.
This example shows the execution order of the controller constructor and its lifecycle hooks: __loader, __init, __rend, __postrend, and __destroy.
Open the browser console to see the constructor and hook arguments.
  1. constructor(app) - Executed when the browser page is reloaded.
  2. __loader(trx) - Loads views via dd-view, loads CSS and JS files, and sets the page title, keywords, description, etc.
  3. __init(trx) - A good place to initialize controller variables and fetch API data using this.$httpClient.
  4. __rend(trx) - This method should not be overridden unless you need to optimize rendering performance.
  5. __postrend(trx) - Runs after the HTML is rendered and the DOM is ready.
  6. __destroy(trx) - Called when a new route is opened and the current controller is destroyed.
app is the App class instance
trx is the router's transition object
