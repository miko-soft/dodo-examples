dd-text

Print pure text.

Destinations: overwrite, remain, append, prepend

dd-text="product.name" → PLACEHOLDER (notice: style will stay on the element unchanged)
dd-text="product.name --overwrite" → PLACEHOLDER
dd-text="product.name --remain" → My dog loves meat
dd-text="product.name --append" → product.name:
dd-text="product.name --prepend" → is the product name


this.

If prefix this. is used it will not cause the error.
dd-text="this.product.name --overwrite" → PLACEHOLDER


HTML

The HTML tags will not be parsed.
dd-text="textWithHtml" →


Pipe option --pipe:

The pipe is any JS String function.
dd-text="longText"→
dd-text="longText --pipe:slice(0, 10).replace(/ipsu/i, 'IPSU').trim()" →
dd-text="longText --pipe:replace(/Lorem/ig, 'LOREM')" →


Automatic data type conversion:

The data type will be automatically converted to string: object → JSON string, boolean → string, number → string
dd-text="obj4json" convert object to JSON string →
dd-text="bool" convert boolean to string →


Dynamic Controller Property Name

When the app requires dynamic controller property name in the dd-text directive use mustaches {{}}
dd-text="product_{{this.pid}}" →
dd-text="product_{{this.pid + 1}}" →
dd-text="product_{{this.pid + 1}}_{{this.pid2}}" →
dd-text="product_{{Math.ceil(this.pid / 2)}}" →


Initial value is undefined

If the initial controller's value is undefined or null the element will not be cloned.

dd-text="$model.job.country" → StartINNER TEXTEnd
dd-text="$model.job.country" --overwrite → StartINNER TEXTEnd
dd-text="$model.job.country" --remain → StartINNER TEXTEnd
dd-text="$model.job.country" --append → StartINNER TEXTEnd
dd-text="$model.job.country" --prepend → StartINNER TEXTEnd
