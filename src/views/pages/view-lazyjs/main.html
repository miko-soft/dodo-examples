Page#lazyJS() Test

The example loads "jQuery", "Notify" and "Sweetalert" libraries lazyly i.e. after all html is rendered and the DOM is ready.
NOTICE: The unlazy will remove SCRIPT tags but it will not unload JS file from the browser memory. So it will stay active.



- will not work unless the "LAZY LOAD LIBS" is clicked

- load jQuery, Notify and Sweetalert libs after waitMS miliseconds

- remove all SCRIPT tags with data-dd-lazyjs (the scripts are still in the browser memory)

- remove the SCRIPT tag which loads Sweetalert (the script is still in the browser memory)

unlayzy JS in destroy hook

We can put "this.unlazyAllJS();" in the destroy hook if we want to unload JS files which are not needed on other pages. Actually it will remove JS files from DOM but it will be still active in browser's memory. If you want to remove it from the memory do something like this, for example in jQuery: "window.$ = undefined;"
To test this, click on /controller-hooks and watch Elements in Chrome DevTools.
