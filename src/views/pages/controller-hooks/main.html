Controller Lifecycle Hooks

Every time a new route is opened the controller is changed and new controller lifecycle starts.
This example shows in which order will be executed controller constructor and the lifecycle hooks. Open the browser console to see the constructor and hook arguments.
  1. constructor(app) - Will be executed on the browser page reload.
  2. loader(trx) - Load views via data-rg-inc and/or data-rg-view.
  3. init(trx) - The place where AJAX calls should be used.
  4. rend(trx) - This method shouldn't be used unless you want to improve the app speed.
  5. postrend(trx) - Render HTML elements after AJAX and render are done.
  6. destroy(trx) - Activated when the route is changed i.e. address in the browser bar.

app is the App class instance
trx is the regoch router transitional variable.
